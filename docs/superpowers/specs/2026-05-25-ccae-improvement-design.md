# CCAE 全面改进设计方案

> 版本: 1.0 | 日期: 2026-05-25 | 状态: 已批准

---

## 一、项目概述

### 1.1 目标
对 CCAE 跨文化适配引擎进行全面改进，包括：
- AI 配置体验重构（参考 CC Switch 设计理念）
- 数据可视化增强
- 系统功能完善
- Bug 修复与性能优化

### 1.2 设计原则
- **零侵入**：配置修改不影响原始文件
- **统一抽象**：多供应商差异封装，用户无感知切换
- **渐进增强**：保持现有功能稳定，增量添加新特性

## 二、AI 配置重构

### 2.1 布局设计
采用 **左侧列表 + 右侧详情** 布局：
- 左侧：供应商列表（可折叠、可排序、支持添加）
- 右侧：当前选中供应商的配置详情

### 2.2 数据模型

```python
# 供应商配置
class ProviderConfig:
    id: str                    # 供应商ID
    name: str                  # 显示名称
    provider_type: str         # 类型: openai/deepseek/qwen/custom
    endpoints: List[Endpoint]  # 端点列表
    is_active: bool            # 是否当前使用

# 端点配置
class Endpoint:
    id: str                    # 端点ID
    name: str                  # 显示名称（如"官方API"、"国内中转"）
    base_url: str              # API端点地址
    api_key: str               # API密钥
    model_mapping: ModelMapping # 模型映射
    is_default: bool           # 是否默认端点
    status: str                # 状态: online/offline/unknown

# 4层模型映射
class ModelMapping:
    primary: str      # 主模型（日常使用）
    light: str        # 轻量模型（快速响应）
    balanced: str     # 均衡模型（质量优先）
    strongest: str    # 最强模型（复杂推理）
```

### 2.3 预设配置

| 供应商 | 默认端点 | primary | light | balanced | strongest |
|--------|----------|---------|-------|----------|-----------|
| OpenAI | 官方 | gpt-4o-mini | gpt-3.5-turbo | gpt-4o | gpt-4o |
| DeepSeek | 官方 | deepseek-chat | deepseek-chat | deepseek-pro | deepseek-reasoner |
| 通义千问 | 官方 | qwen-turbo | qwen-turbo | qwen-plus | qwen-max |

### 2.4 界面功能

**供应商列表：**
- 显示供应商名称 + 状态指示器
- 点击切换当前供应商
- 右键菜单：编辑、删除、测试连接
- 底部"添加供应商"按钮

**配置详情区：**
- 基本信息：名称、类型
- 端点管理：端点列表 + 添加/编辑/删除
- 模型映射：4层模型配置
- 操作按钮：保存、测试连接、设为当前

### 2.5 API 设计

```
GET  /api/ai/providers          # 获取所有供应商配置
POST /api/ai/providers          # 添加供应商
PUT  /api/ai/providers/:id      # 更新供应商配置
DELETE /api/ai/providers/:id    # 删除供应商

POST /api/ai/providers/:id/activate  # 激活供应商
POST /api/ai/providers/:id/test      # 测试连接

GET  /api/ai/current            # 获取当前激活配置
```

## 五、Bug 修复与优化

### 5.1 已知问题修复

1. **编码问题**：Windows 终端中文输出乱码
   - 解决：使用 `sys.stdout.reconfigure(encoding='utf-8')`

2. **.env 读取**：部分环境下读取失败
   - 解决：添加 python-dotenv 依赖

3. **图片路径验证**：边界情况处理不完整
   - 解决：完善路径验证逻辑

### 5.2 性能优化

1. **API 响应缓存**：对不常变化的数据添加缓存
2. **表格分页**：大数据量时使用虚拟滚动
3. **前端防抖**：搜索输入添加 debounce

### 5.3 用户体验优化

1. **加载状态**：所有异步操作显示 loading
2. **错误提示**：友好的错误信息展示
3. **操作确认**：危险操作添加确认对话框

## 七、技术栈

- **后端**：Python Flask + SQLite（保持不变）
